The Next Block is a free AI prompt that refuses to let a goal stay vague.

See it in action

The interview narrows the work. The file holds the block. The log points to the next one.

1. The interview refuses vagueness

The prompt does not accept the fuzzy version. It keeps asking until the work becomes one concrete block.

User"Work on my book."

AI"That's a project, not a block. Which scene, and what physical action starts it?"

User"Reread the last page of Caves, then draft the descent — 9:00 to 10:30."

User"Work on my story."

AI"That's too vague. Which scene? What exactly will be done by the end of the block?"

User"Revise the opening scene."

AI"Still broad. I'd make the block: reread the opening once, cut the throat-clearing paragraph, and rewrite the last six lines so the scene lands. Edit these before I build the file."

2. The file holds the block

After the interview, Claude builds a simple offline file: action list, reason, start time, check-ins, and a local log.
